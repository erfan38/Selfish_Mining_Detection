

# Efficient Detection of Selfish Mining Attacks Using Machine Learning

> **Efficient Detection of Selfish Mining Attacks on Large-Scale Blockchain Networks.**
> *2024 IEEE 24th International Conference on Software Quality, Reliability, and Security Companion (QRS-C)*, July 2024.

## Overview

**Selfish mining** is a strategic attack in which a malicious miner (or pool) secretly extends a private chain rather than immediately broadcasting mined blocks, gaining a disproportionate share of rewards at the expense of honest miners. Detecting this attack at scale is difficult because selfish behavior is statistically subtle and many network features are artifacts of specific simulators rather than observable in real networks.

This work proposes a **machine-learning-based detection pipeline** combining **Principal Component Analysis (PCA)** for dimensionality reduction and **Random Forest Classification (RFC)** for detection. We:

1. Validate the approach on an **existing dataset** from the ForkDec paper.
2. **Generate a new dataset** using a modified Bitcoin simulator to produce richer, more realistic features.
3. **Compare performance** against prior work and show improvements in detection efficiency.

A key contribution is a **feature-selection analysis** that identifies which network-level features are actually useful for selfish-mining detection in a real Bitcoin network (as opposed to being simulator-specific artifacts).

## Datasets

### Dataset 0 — Full 32-feature simulated dataset

Gathered from a simulated Bitcoin network and containing **32 features**. All are related to selfish mining in principle, but many are dependent on simulator-specific parameters and would be unavailable or unreliable in a real network. We analyzed all 32 and identified only **6 features independent of simulator internals** and genuinely useful for real-world detection.

### Dataset 1 — ForkDec dataset (existing)

The existing dataset introduced in the ForkDec paper. Used for validating the approach against prior work. Files: `training.csv`, `testing.csv`.

### Dataset 2 — Our 6-feature simulated dataset

Generated by modifying the [Bitcoin Simulator by Arthur Gervais](https://github.com/arthurgervais/Bitcoin-Simulator) and extracting the 6 independent features identified from Dataset 0. File: `shuffle_6f.csv`.

### Dataset 3 — Single-feature dataset

Contains only the single most informative feature for selfish-mining detection. File: `shuffle_data1_receivedTime.csv`.

## Approach

- **Preprocessing** — cleaning, normalization, and feature selection for each dataset.
- **PCA** — dimensionality reduction applied consistently across training and testing splits using the same fitted PCA model.
- **Classification** — Random Forest Classifier (RFC) as the primary detector.
- **Deep-learning baseline** — a Fully Connected Neural Network (FC-NN) is also evaluated on Datasets 2 and 3 for comparison.
